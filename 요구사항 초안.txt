​🚀 [초정밀 요구사항] AI UI 에이전트 엔터프라이즈 워크벤치 구축
​[시스템 메인 가이드라인]
• ​불변의 레이아웃: 반드시 3단 분할 레이아웃을 고정하십시오. (좌측 22% 트리 | 중앙 38% 에디터 및 컨트롤 | 우측 40% 프리뷰)
• ​자율 수정 금지: 사용자가 명시적으로 요청하지 않는 한, 이 명세서에 정의된 UI 구조나 버튼 위치, 핵심 로직을 임의로 변경하거나 생략하지 마십시오.
​1. 좌측 영역: 프로젝트 트리 (Project Tree)
• ​위치 및 디자인: 화면 좌측 22%, 화이트 배경, 우측 경계선(slate-200), 상단에 'Project Tree' 타이틀과 '1-2-3 Hierarchy' 라벨 노출.
• ​계층 구조 (1-2-3 Depth): 
• ​1-Depth: 프로젝트 루트 (예: '마이영역 기획').
• ​2-Depth: 시나리오의 큰 번호 (예: '1. 상단 영역', '2. 중앙 콘텐츠'). [중요] 2단계는 하위 UI를 조립해 보여주는 그룹 역할입니다.
• ​3-Depth: 시나리오의 세부 번호 (예: '1-1. 네비게이션', '2-1. 페이 모듈'). [중요] 실제 UI가 생성되는 최소 단위입니다.
• ​기능: 
• ​모든 트리 항목 우측에 **연필 아이콘(명칭 수정)**과 삭제 아이콘 배치.
• ​항목 클릭 시 중앙 에디터와 우측 프리뷰가 해당 노드의 데이터로 즉시 전환.
• ​렌더링 여부에 따라 좌측에 초록색(완료) / 회색(미완료) 상태 표시등 노출.
​2. 중앙 영역: 워크스페이스 (Workspace)
• ​상단 헤더: 현재 선택된 메뉴 이름과 버전(v1.0 등) 표시. 우측에 [기획 최종 확정] 버튼 배치.
• ​파이프라인 컨트롤 박스 (Header 하단): 
• ​배경색: 연한 블루(blue-50), 테두리 강조.
• ​[1단계: 트리 분석 및 상세 시나리오 분할] 버튼: 전체 시나리오를 1-2-3 댑스로 쪼개고, 3단계 노드에 설명 원문을 생략 없이 100% 매핑.
• ​[2단계: 일괄 UI 조립]:  나 2단계 선택 시 활성화. 하위의 모든 3단계 모듈을 순차적으로 하나씩 API 호출하여 렌더링.
• ​[2단계: 현재 모듈 UI 렌더링]: 3단계 단일 노드 선택 시 활성화.
• ​시나리오 에디터: 하단 나머지 영역 전체. 흰색 배경의 대형 textarea. 자동 저장 기능 필수.
​3. 우측 영역: 프리뷰 및 분석 (Preview & Analysis)
• ​상단 버튼 바: 프리뷰 상단에 [FLOW] [DIAGRAM] [CODE] 버튼을 가로로 배치.
• 기본적으로 프리뷰 상태이며 각 버튼 클릭시 api호출하여 데이터 노출함
• Flow - 비즈니스 FLOW 노출
• DIAGRAM - mermaid.js를 활용하여 다이어그램 노출
• code - 수정필요한 코드에 대한 리스트 노출
• ​모바일 프리뷰 (Phone Frame): 
• ​위치: 화면 중앙 정렬, iPhone 스타일 프레임.
• ​Gap-Zero 조립 엔진: 1/2단계 클릭 시 하위 3단계 UI 조각들을 합치되, margin:0, padding:0, line-height:0을 강제하여 모듈 간 빈 공간을 완벽 제거.
• ​스마트 캐싱: 시나리오 텍스트가 이전 렌더링 시점과 동일하면 API를 호출하지 않고 기존 HTML 재사용.
• ​스크롤: 프레임 내부에서 독립적인 세로 스크롤 가능.
• ​분석 모달 (Popup): 
• ​확정 후 하이라이트: v1.0 확정 이후 시나리오 수정 시, FLOW와 DIAGRAM 노드 중 변경된 부분만 주황색(#F97316)으로 하이라이트.
• ​DIAGRAM: Mermaid JS 사용. [필수] 신택스 에러 방지를 위해 모든 노드 라벨은 반드시 큰따옴표("")로 감쌀 것.
• ​CODE: 변경이 필요한 소스 파일명(JAVA, VUE, SQL)과 수정 내용 가이드 리스트 출력.
​4. 기술적 디테일 (Technical Requirements)
• ​Persistence: 모든 데이터(트리, 시나리오, HTML, 확정 상태)를 LocalStorage에 실시간 저장.
• ​CSS Isolation: 프리뷰 영역에 isolation: isolate를 적용하고, 생성된 코드의 fixed 속성은 프레임 내부의 absolute로 작동하도록 강제.
• ​Error Handling: API 호출 실패 시 Exponential Backoff 재시도 로직 적용 및 토스트 알림 표시.
